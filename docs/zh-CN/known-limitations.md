# Known Limitations

[English](../known-limitations.md) | [文档](README.md)

Papyro 目前适合开发自测和 dogfooding，但还不是完整的终端用户正式版。这个页面记录当前已知限制，帮助贡献者和测试者区分“预期内粗糙点”和“需要立即修复的问题”。

## 产品状态

- 当前主要目标是桌面端。
- 移动端已经共享 runtime 和资源，但还不是生产可用状态。
- 基础桌面端 zip packaging 已存在；原生安装器还没有完成。
- 当前只做本地优先；云同步、账号和协作暂时不在范围内。

## Markdown 编辑

- Hybrid 模式仍然对架构很敏感。光标命中、选区行为、inline decoration 边界、IME 和键盘导航还需要继续打磨，才能称为企业级体验。
- 如果需要精确控制 Markdown，Source 和 Preview 目前更稳。
- Mermaid、数学公式、表格、链接、图片和代码块已经支持，但编辑入口和交互仍在优化。
- 大文档可能会暂停实时预览或代码高亮，以保持编辑响应速度。
- Markdown 中的 raw HTML 会被清洗，不会作为任意 HTML 渲染。

## Workspace 和文件

- 依赖大型 workspace 做日常工作前，建议先用真实目录测试。
- Workspace 会跟随当前 active tab 切换，但完整多窗口路由还没有实现。
- OS 文件关联正在推进；安装器级别的注册还没有完成。
- 回收站和恢复草稿已经存在，但重要笔记仍建议保留外部备份。

## 桌面壳层

- 设置仍然运行在主窗口体验中。后续计划改成进程级工具窗口。
- core 中已经建模多窗口文档编辑，但还不是用户可见模式。
- 仍有一些发布级打磨未完成：原生安装器和演示素材。

## 性能

- 自动化 smoke 覆盖了高风险路径，但大改 editor 或壳层前仍需要采集手工桌面端 trace。
- 编辑器 bundle 是本地离线资源，但修改 `js/src` 后必须同步生成的 JS 资源。

## 应该报告什么

请优先报告比上述限制更严重的问题，尤其是：

- 数据丢失、保存失败恢复异常，或 dirty 状态让人困惑
- 可复现的光标、选区、粘贴或 IME 问题
- 文件操作阻塞 UI
- workspace 扫描在真实项目上卡住
- 暗色模式对比度或窄窗口布局回归
